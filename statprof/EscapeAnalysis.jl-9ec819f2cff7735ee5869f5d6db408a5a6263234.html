41 """
42 x::EscapeInfo
43
44 A lattice for escape information, which holds the following properties:
45 - `x.Analyzed::Bool`: not formally part of the lattice, only indicates whether `x` has been analyzed
46 - `x.ReturnEscape::Bool`: indicates `x` can escape to the caller via return
47 - `x.ThrownEscape::BitSet`: records SSA statement numbers where `x` can be thrown as exception:
48 * `isempty(x.ThrownEscape)`: `x` will never be thrown in this call frame (the bottom)
49 * `pc ∈ x.ThrownEscape`: `x` may be thrown at the SSA statement at `pc`
50 * `-1 ∈ x.ThrownEscape`: `x` may be thrown at arbitrary points of this call frame (the top)
51 This information will be used by `escape_exception!` to propagate potential escapes via exception.
52 - `x.AliasInfo::Union{Bool,IndexableFields,IndexableElements,Unindexable}`: maintains all possible values
53 that can be aliased to fields or array elements of `x`:
54 * `x.AliasInfo === false` indicates the fields/elements of `x` aren't analyzed yet
55 * `x.AliasInfo === true` indicates the fields/elements of `x` can't be analyzed,
56 e.g. the type of `x` is not known or is not concrete and thus its fields/elements
57 can't be known precisely
58 * `x.AliasInfo::IndexableFields` records all the possible values that can be aliased to fields of object `x` with precise index information
59 * `x.AliasInfo::IndexableElements` records all the possible values that can be aliased to elements of array `x` with precise index information
60 * `x.AliasInfo::Unindexable` records all the possible values that can be aliased to fields/elements of `x` without precise index information
61 - `x.Liveness::BitSet`: records SSA statement numbers where `x` should be live, e.g.
62 to be used as a call argument, to be returned to a caller, or preserved for `:foreigncall`:
63 * `isempty(x.Liveness)`: `x` is never be used in this call frame (the bottom)
64 * `0 ∈ x.Liveness` also has the special meaning that it's a call argument of the currently
65 analyzed call frame (and thus it's visible from the caller immediately).
66 * `pc ∈ x.Liveness`: `x` may be used at the SSA statement at `pc`
67 * `-1 ∈ x.Liveness`: `x` may be used at arbitrary points of this call frame (the top)
68
69 There are utility constructors to create common `EscapeInfo`s, e.g.,
70 - `NoEscape()`: the bottom(-like) element of this lattice, meaning it won't escape to anywhere
71 - `AllEscape()`: the topmost element of this lattice, meaning it will escape to everywhere
72
73 `analyze_escapes` will transition these elements from the bottom to the top,
74 in the same direction as Julia's native type inference routine.
75 An abstract state will be initialized with the bottom(-like) elements:
76 - the call arguments are initialized as `ArgEscape()`, whose `Liveness` property includes `0`
77 to indicate that it is passed as a call argument and visible from a caller immediately
78 - the other states are initialized as `NotAnalyzed()`, which is a special lattice element that
79 is slightly lower than `NoEscape`, but at the same time doesn't represent any meaning
80 other than it's not analyzed yet (thus it's not formally part of the lattice)
