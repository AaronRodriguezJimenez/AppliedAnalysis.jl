3 """
4 NamedTuple
5
6 `NamedTuple`s are, as their name suggests, named [`Tuple`](@ref)s. That is, they're a
7 tuple-like collection of values, where each entry has a unique name, represented as a
8 [`Symbol`](@ref). Like `Tuple`s, `NamedTuple`s are immutable; neither the names nor the values
9 can be modified in place after construction.
10
11 A named tuple can be created as a tuple literal with keys, e.g. `(a=1, b=2)`,
12 or as a tuple literal with semicolon after the opening parenthesis, e.g. `(;
13 a=1, b=2)` (this form also accepts programmatically generated names as
14 described below), or using a `NamedTuple` type as constructor, e.g.
15 `NamedTuple{(:a, :b)}((1,2))`.
16
17 Accessing the value associated with a name in a named tuple can be done using field
18 access syntax, e.g. `x.a`, or using [`getindex`](@ref), e.g. `x[:a]` or `x[(:a, :b)]`.
19 A tuple of the names can be obtained using [`keys`](@ref), and a tuple of the values
20 can be obtained using [`values`](@ref).
21
22 !!! note
23 Iteration over `NamedTuple`s produces the *values* without the names. (See example
24 below.) To iterate over the name-value pairs, use the [`pairs`](@ref) function.
25
26 The [`@NamedTuple`](@ref) macro can be used for conveniently declaring `NamedTuple` types.
27
28 # Examples
29 ```jldoctest
30 julia> x = (a=1, b=2)
31 (a = 1, b = 2)
32
33 julia> x.a
34 1
35
36 julia> x[:a]
37 1
38
39 julia> x[(:a,)]
40 (a = 1,)
41
42 julia> keys(x)
43 (:a, :b)
44
45 julia> values(x)
46 (1, 2)
47
48 julia> collect(x)
49 2-element Vector{Int64}:
50 1
51 2
52
53 julia> collect(pairs(x))
54 2-element Vector{Pair{Symbol, Int64}}:
55 :a => 1
56 :b => 2
57 ```
58
59 In a similar fashion as to how one can define keyword arguments programmatically,
60 a named tuple can be created by giving pairs `name::Symbol => value` after a
61 semicolon inside a tuple literal. This and the `name=value` syntax can be mixed:
62
63 ```jldoctest
64 julia> (; :a => 1, :b => 2, c=3)
65 (a = 1, b = 2, c = 3)
66 ```
67
68 The name-value pairs can also be provided by splatting a named tuple or any
69 iterator that yields two-value collections holding each a symbol as first
70 value:
71
72 ```jldoctest
73 julia> keys = (:a, :b, :c); values = (1, 2, 3);
74
75 julia> NamedTuple{keys}(values)
76 (a = 1, b = 2, c = 3)
77
78 julia> (; (keys .=> values)...)
79 (a = 1, b = 2, c = 3)
80
81 julia> nt1 = (a=1, b=2);
82
83 julia> nt2 = (c=3, d=4);
84
85 julia> (; nt1..., nt2..., b=20) # the final b overwrites the value from nt1
86 (a = 1, b = 20, c = 3, d = 4)
87
88 julia> (; zip(keys, values)...) # zip yields tuples such as (:a, 1)
89 (a = 1, b = 2, c = 3)
90 ```
91
92 As in keyword arguments, identifiers and dot expressions imply names:
93
94 ```jldoctest
95 julia> x = 0
96 0
97
98 julia> t = (; x)
99 (x = 0,)
100
101 julia> (; t.x)
102 (x = 0,)
103 ```
104
105 !!! compat "Julia 1.5"
106 Implicit names from identifiers and dot expressions are available as of Julia 1.5.
107
108 !!! compat "Julia 1.7"
109 Use of `getindex` methods with multiple `Symbol`s is available as of Julia 1.7.

309 """
310 merge(a::NamedTuple, bs::NamedTuple...)
311
312 Construct a new named tuple by merging two or more existing ones, in a left-associative
313 manner. Merging proceeds left-to-right, between pairs of named tuples, and so the order of fields
314 present in both the leftmost and rightmost named tuples take the same position as they are found in the
315 leftmost named tuple. However, values are taken from matching fields in the rightmost named tuple that
316 contains that field. Fields present in only the rightmost named tuple of a pair are appended at the end.
317 A fallback is implemented for when only a single named tuple is supplied,
318 with signature `merge(a::NamedTuple)`.
319
320 !!! compat "Julia 1.1"
321 Merging 3 or more `NamedTuple` requires at least Julia 1.1.
322
323 # Examples
324 ```jldoctest
325 julia> merge((a=1, b=2, c=3), (b=4, d=5))
326 (a = 1, b = 4, c = 3, d = 5)
327 ```
328
329 ```jldoctest
330 julia> merge((a=1, b=2), (b=3, c=(d=1,)), (c=(d=2,),))
331 (a = 1, b = 3, c = (d = 2,))
332 ```
333 """
334 function merge(a::NamedTuple{an}, b::NamedTuple{bn}) where {an, bn}
335 if @generated
336 names = merge_names(an, bn)
337 types = merge_types(names, a, b)
338 vals = Any[ :(getfield($(sym_in(names[n], bn) ? :b : :a), $(QuoteNode(names[n])))) for n in 1:length(names) ]
339 :( _new_NamedTuple(NamedTuple{$names,$types}, ($(vals...),)) )
340 else
341 merge_fallback(a, b, an, bn)
342 end
343 end

360 Interpret an iterable of key-value pairs as a named tuple, and perform a merge.
361
362 ```jldoctest
363 julia> merge((a=1, b=2, c=3), [:b=>4, :d=>5])

367 function merge(a::NamedTuple, itr)
368 names = Symbol[]
369 vals = Any[]
370 inds = IdDict{Symbol,Int}()
371 for (k, v) in itr
372 k = k::Symbol
373 oldind = get(inds, k, 0)
374 if oldind > 0
375 vals[oldind] = v
376 else
377 push!(names, k)
378 push!(vals, v)
379 inds[k] = length(names)
380 end
381 end
382 merge(a, NamedTuple{(names...,)}((vals...,)))

472 """
473 @NamedTuple{key1::Type1, key2::Type2, ...}
474 @NamedTuple begin key1::Type1; key2::Type2; ...; end
475
476 This macro gives a more convenient syntax for declaring `NamedTuple` types. It returns a `NamedTuple`
477 type with the given keys and types, equivalent to `NamedTuple{(:key1, :key2, ...), Tuple{Type1,Type2,...}}`.
478 If the `::Type` declaration is omitted, it is taken to be `Any`. The `begin ... end` form allows the
479 declarations to be split across multiple lines (similar to a `struct` declaration), but is otherwise
480 equivalent. The `NamedTuple` macro is used when printing `NamedTuple` types to e.g. the REPL.
481
482 For example, the tuple `(a=3.1, b="hello")` has a type `NamedTuple{(:a, :b), Tuple{Float64, String}}`, which
483 can also be declared via `@NamedTuple` as:
484
485 ```jldoctest
486 julia> @NamedTuple{a::Float64, b::String}
487 @NamedTuple{a::Float64, b::String}
488
489 julia> @NamedTuple begin
490 a::Float64
491 b::String
492 end
493 @NamedTuple{a::Float64, b::String}
494 ```
495
496 !!! compat "Julia 1.5"
497 This macro is available as of Julia 1.5.
498 """
499 macro NamedTuple(ex)
500 Meta.isexpr(ex, :braces) || Meta.isexpr(ex, :block) ||
501 throw(ArgumentError("@NamedTuple expects {...} or begin...end"))
502 decls = filter(e -> !(e isa LineNumberNode), ex.args)
503 all(e -> e isa Symbol || Meta.isexpr(e, :(::)), decls) ||
504 throw(ArgumentError("@NamedTuple must contain a sequence of name or name::type expressions"))
505 vars = [QuoteNode(e isa Symbol ? e : e.args[1]) for e in decls]
506 types = [esc(e isa Symbol ? :Any : e.args[2]) for e in decls]
507 return :(NamedTuple{($(vars...),), Tuple{$(types...)}})
508 end
